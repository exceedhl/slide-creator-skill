# Visual Style: McKinsey / Swiss Style (麦肯锡 / 瑞士风格)

这是生成幻灯片的默认视觉风格。所有未被组件 `references/components/*.md` 覆盖的视觉决策（色值、字号、阴影、圆角等），均以本文件为唯一权威来源。

## 1. 视觉哲学

* 瑞士风格 (Swiss Style)：以网格、秩序、功能性为核心美学
* 单色调为主 (Monochromatic)：以冷灰-深蓝为基底，极度克制使用彩色
* 高信息密度 (High-Density)：拒绝空洞留白。正文字号饱满，卡片紧凑，依靠物理填充率强化专业厚重感
* 极简主义 (Minimalism)：去装饰，让数据与结论成为视觉焦点

## 2. 色彩系统与层级字典 (Color Hierarchy)

本套体系完全摒弃花哨的彩色，依靠冷色调的对比度和层级来界定信息的轻重缓急。**严禁在正文使用弱灰色，必须确保极高的阅读对比度。**

### 2.1 核心主调 (Core & Accent)

| 变量名 | 色值 | 用途与约束 |
|--------|------|------|
| `--primary` | `#0F172A` | **主视觉锚点**：大标题、最核心数据、基底元素描边。 |
| `--primary-light` | `#1E40AF` | **次主要视觉系列**：图表次维度填充，或强调性组件的辅助着色。 |
| `--accent` | `#EA580C` | **视觉刺客 (强调色)**：极度克制。仅在最核心聚焦节点、Active 状态使用。严禁大面积铺底。 |

### 2.2 表面与底色 (Surface Levels)

遵循严格的明暗 Z-Index 层级推进：
* L0 根底板：`white (#FFFFFF)`
* L1 辅助底板 `--bg-surface-light`：`#F8FAFC`
* L2 核心卡片表面 `--bg-surface`：`#F1F5F9`（用于装载正文、象限矩阵、流程节点）
* L3 交替隔离块 `--bg-surface-alt`：`#E2E8F0`（表格交替行、次级块围栏）

### 2.3 文本与骨架 (Text & Utility)

| 变量名 | 色值 | 用途与约束 |
|--------|------|------|
| `--text-muted` | `#0F172A` | **高对比正文军团**：用于海量正文段落。**为保证信息密度与干练，必须采用近似纯黑的深色，绝不可使用浅灰。** *(此项为最新演进规范，替代陈旧的 #64748B)* |
| `--text-dim` | `#94A3B8` | **隐性退让信息**：最底部的 Source 脚注、背景大号编号。 |
| `--border-default` | `#E2E8F0` | **物理隔断骨架**：用于卡片边框、表格 1px 细线。 |

## 3. 排版规范 (Typography)

### 3.1 字体族 (Font Stack)

为了强化商业咨询报告的权威感与书面严肃性，中文排版需优先使用**楷体系列**。
核心映射逻辑：`font-family: "Helvetica Neue", Arial, "Kaiti SC", Kaiti, STKaiti, serif;`
（英文字符保持无衬线体，中文字符强制回退至楷体）。

### 3.2 字号层级表与呼吸感控制

为了实现**减少留白、强化密度**的策略，基础字号与间距必须被严格收紧：

* 页面底盘边距：由宽泛的 `50px 60px` 缩减为 **`32px 40px`** 左右
* 组件间隙 (`--gap`)：统一缩减至 **`10px ~ 16px`**，坚决消灭过于松散的版面感

| 角色 | 字号规范 | 字重 | 颜色规范 |
|------|------|------|------|
| 页面大标题 (h1) | 26-28px | 700 | `--primary` |
| 副标题 (h2) | 16px | 400 | `--text-muted` |
| 卡片/模块标题 (h3/h4)| 16-20px | 700 | `--primary` |
| 正文 (body/p) | 14-16px | 400 | `--text-muted` |
| 数据标注 (data) | 24-32px | 700 | `--primary` 或 `--border-default` (做背景级数字) |
| 脚注/Source | 11px | 400 | `--text-dim` |

*注：避免将大块的正文设为 12px 或 13px 等过弱的字号。在呈现高密度的数据表格时，字号应向下克制于 `14px`。*

## 4. 全局视觉基建 (Global Primitives)

### 4.1 表面与卡片容器 (Surfaces & Cards)
* **绝对平面化**：内部卡片**无边框、无阴影**，单纯依靠背景色区隔，这是极其重要的瑞士极简基因。

### 4.2 线条 (Lines)
* **分割线统一降噪**：大标题下方允许使用 2px 线条，其余所有局部隔断、表格连线强制缩为 **1px 细实线**。

## 5. 组件级核心铁律 (Component Specifics)

### 5.1 数据表格 (TABLE)
* **极简横流线框**：仅依靠贯通全宽的**横向 1px 细实线**区隔。严禁使用顶部封顶线及任何垂直列的纵向分割线。

### 5.2 标题与脚注 (Title & Source)
* **底层锚定**：脚注 Source 必须不可动摇地锚定在网格系统最底部（如 `grid-row: 24`）。严禁编造任何虚假的数据来源。

### 5.3 左右分割与全通栏 (Split Layout & Full Bleed)
* **边缘突破**：作为环境调性的一侧（如大面积配图或重色块区块）必须突破 padding 紧贴画布物理边缘（Full Bleed）。相对的文字陈述侧则必须保留极其克制的安全边距（Safe Area），形成“盈满与留白”的极致视觉张力对抗。

## 6. 图标使用规范 (Icon Usage Guidelines)

麦肯锡 / 瑞士风格中，图标的作用是**极简辅助认知**而非花哨装饰。必须保持极度克制、绝对扁平和语义对齐。

### 6.1 图标来源：Lucide Icons

统一强制使用 [Lucide Icons](https://lucide.dev) 作为图标源。该库的极简线性设计与此页面的学术网格美学有着绝佳的契合度。

**引入声明方式**：通过无依赖 CDN 引入，在 HTML 中使用 `data-lucide` 属性声明图标：

```html
<!-- 在 </body> 前优先引入 -->
<script src="https://unpkg.com/lucide@latest"></script>
<script>lucide.createIcons();</script>

<!-- 在 HTML 节点声明 -->
<i data-lucide="trending-up"></i>
<i data-lucide="check-circle"></i>
```

### 6.2 线性与克制铁律

* **极简线条权重（1.5px/2px）**：严格利用内部属性或 CSS（如 `stroke-width: 1.5`）强制约束图标线段粗细为 1.5 到 2，这能极大削减视觉笨重感。**绝对严禁掺杂多色、3D立体、拟物化或加持任何阴影的图标**。
* **单色调纯净染色**：图标的颜色必须严格绑定至系统的 `--primary` 变量，次要状态为 `--text-muted`。全局绝不允许出现第三种图标高亮色。同一个卡片矩阵内的所有图标必须保持色系统一。
* **尺寸约束机制**：默认 24px。若作为列表项内联标识，行内图标需克制缩小至 `18-20px`。作为卡片核心图腾的独立图标框，最大极限允许放至 `32px`。

### 6.3 辅助重力图腾与排他红线

* **底色数字徽章**：深色直角小方块内嵌数字（`01`, `02`...）。这几乎是此类分析报告中最具威势的压迫性序号段。使用 `--primary` 或 `--accent` 铺底色 + 纯白文字，**严禁注入圆角**。
* **全局排他**：❌ 坚决抵制、绝对禁止引入 Font Awesome、Material Icons、Griddy Icons 等带有实心面积块或原生风格冲突的杂牌第三方图库，混合使用将导致顶级商业质感全盘崩溃。
