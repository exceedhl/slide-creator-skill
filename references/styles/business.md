# Visual Style: Business Style (商务风格)

这是基于专业化、商业报告的视觉风格。所有未被组件 `references/components/*.md` 覆盖的视觉决策（色值、字号、阴影、圆角等），应以此为基准，确保在正式的商业演示中传达权威、极简与专业的数据呈现。

## 1. 视觉哲学

* **商务与专业 (Professional & Trustworthy)**：以深沉的商务蓝（Deep Blue）传递稳定感与权威性。
* **绝对直角的严肃美学 (Sharp & Formal)**：全场零圆角（`border-radius: 0`），抛弃现代 App 常见的圆润感，塑造顶级商业咨询（如麦肯锡风）特有的严谨与硬朗体系。
* **克制的面状分割 (Restrained Surfaces)**：去边框化。全面抛弃生硬的实线边框，纯粹依靠克制的浅色背景区块（如浅灰蓝区域底色）确立物理隔断，既保证内容成团，又减少视觉压迫。
* **排版满刻度与防窒息 (Density vs. Breathing)**：通过系统性放大大字号体系以消除空旷感（减少留白），但同时将大规模落地的正文颜色调浅为深灰色（减轻纯黑色的窒息感），仅保留重点文本为纯黑色以拉满对比度。

## 2. 色彩系统与层级字典 (Color Hierarchy)

本套体系以蓝色系为核心，具备高度的统一感与极强的阅读层级感。

### 2.1 核心主调 (Core & Accent)

| 变量名 | 色值 | 用途与约束 |
|--------|------|------|
| `--primary` | `#051C2C` | **主视觉锚点**：极深的商务蓝。用于页面大标题、重点数据块标题、图表主轴文字及最核心元素的颜色。 |
| `--accent` | `#005EB8` | **核心品牌色/强调色**：经典的商业亮蓝。用于重要的高光块（Highlight box）、数据徽章（Badge）、局部对比性装饰图标等。 |
| `--accent-light` | `#009CE4` | **次级强调色**：明快的浅蓝。配合主色使用，比如封面的渐变光影、次要图标点缀等。 |
| `--chart-red` | `#C93B3B` | **对比点缀色**：用于极少数需要形成强烈对比的负面或竞品数据，极度克制使用。 |

### 2.2 表面与容器底色 (Surface Levels)

遵循稳重且面状的灰白卡片层级，取代冗余的线框。

| 层级与变量 | 色值 | 适用场景 |
|------|------|------|
| L0 全局画布边框 | `#E5E7EB` | 幻灯片的外部基础画布底色。 |
| L1 `--bg-color` | `#FFFFFF` | **幻灯片主底板**：保证高度信息的通透与留白。 |
| L2 `--bg-surface-light` | `#F8FAFC` | **功能性色块底板**：高频用于正文对比列、步骤说明卡片、重点提炼框的内部底色填充，作为无边框物理分割基石。 |
| L3 `--table-header` | `#F1F5F9` | **次级强调底板**：表格表头背景、模型结构说明底板等。 |

### 2.3 文本与骨架 (Text & Utility)

| 变量名 | 色值 | 用途与约束 |
|--------|------|------|
| `--text-main` | `#333333` | **高光强调文本**：用于 `<strong>` 强调标签、最核心数据的数字，与大面积的正文灰形成强烈区隔。 |
| `--text-light` | `#666666` 或 `#52525B` | **陈述正文大军**：所有中长段落主体的字体颜色（`li`, `p`）。强制替换纯黑，极大减轻密集排版的视觉“窒息感”，创造阅读呼吸空间。 |
| `--grid-line` | `#E2E8F0` | **物理构架辅助线**：用于极少情况下的底层分割，如图表坐标辅助轴线、表格水平分割线。 |

## 3. 排版规范 (Typography)

### 3.1 字体族 (Font Stack)

强制优先采用 `"Helvetica Neue", Arial, sans-serif`。字体设计必须无衬线，传达现代与干练的宏观感受。

### 3.2 字号层级表 (Type Scale)

为了解决自动化宽屏展示易出现的“空虚感”，全局字号需要维持**大而饱满**。所有样式禁止使用偏小的字号：

| 角色 | 推荐字号 | 推荐字重 | 颜色运用 |
|------|------|------|------|
| 封面主标题 (Cover h1) | 52-54px | 700/800 | `#FFFFFF` (配合深色面) |
| 内页主标题 (h2) | 38-40px | 800 | `--primary` |
| 内页副标题 (h3/subtitle) | 22-26px | 400 | `--text-light`（浅灰） |
| 功能区块标题 (列/卡片 h3) | 22-28px | 700 | `--primary` 或 `--accent` |
| 正文大军 (body/li) | 16-17px | 400 | `--text-light` |
| 重点字段标黑 (`<strong>`) | 17-18px | 700 | `--text-main` 或 `--primary` |
| 脚注/Source | 13px | 400 | `--text-light` |

### 3.3 行高与间距

* 标题区：`line-height: 1.25 ~ 1.4`
* 正文内容：`line-height: 1.5 ~ 1.6`（即使版面紧凑，正文内部也必须留够呼吸感）
* 组内聚：保证区块之间使用 `20px ~ 24px` 的内边距将文本向内挤压以聚拢内容。

## 4. 全局视觉基建 (Global Primitives)

### 4.1 边角与边框纪律 (Corners & Borders)

* **严禁圆角**：整个报告体系中（包括图表元素、对比色块、背景徽章框），绝对禁止包含任何 `border-radius`，必须是锋利克制的直角 `0px`。
* **严禁描边封装**：在卡片和组快层，**禁止使用 `1px solid var(--grid-line)` 描边**。模块分割纯靠色相变化（如直接填充 `--bg-surface-light` 且无描边）与白边距。
* **阴影豁免**：极少且克制地对完全悬浮的核心图腾（如圆形的 VS 居中图表）可用阴影。正文卡片 `100%` 摒弃盒阴影（box-shadow）。

### 4.2 极简的视觉隐喻点缀 (Micro-Graphic Decorators)

当大面积文字易造成同构疲劳时，必须引入局部图腾或背景辅助修饰：
* **结构化打断符**：在两栏对立对比时，可在中间使用反白的对比组件（例如深蓝色宽大的全圆 `VS` 切割线或图标），通过其几何张力破开均分的布局留白。
* **超大数字水痕**：在并列步骤（Steps）或阶梯逻辑背后，大量利用伪类 `::after` 铺垫具有冲击力的超大数字水印（如 `font-size: 140px; color: rgba(0,0,0,0.03)`）。通过设置 `height: 0; line-height: 0; overflow: visible;` 黑客级排版脱离常规文本流限制（同时确保通过防溢出边界检查），以此丰富背景层次并提升高级设计感。

### 4.3 色彩纪律 (Anti-Rainbow UI)

* **严禁“为了区分而上色”**：在展现并列层级的信息（如 2x2 矩阵象限、阶梯图、时间轴步骤）时，**绝对禁止**给每个平级模块分配不同的彩色（如主色、亮色、红色混搭）。这会产生极其廉价的“视觉塑料味”。
* **单主色与唯一高亮**：整个页面或组件的主干结构必须用高度克制的柔和灰白或主色统一呈现。**仅针对**最需强调的核心结论施加唯一的加签提醒（例如粗下划线或 `--accent` 点亮），其他平级项全额收敛。

## 5. 组件级说明 (Component Specifics)

### 5.1 标题区与来源脚注

* **标题块底线切断**：主副标题所在的容器区域（Title Block），其底部必须使用 `border-bottom: 2px solid var(--primary);` 与正文内容进行物理切断。严禁错将其写为 `border-top`。
* **来源（Source）**：脚注必须使用灰色或暗色锁定放置于网格最底部（`grid-row: 24`），确保学术及商业严谨性。

### 5.2 交互式列布局与步骤

* 在呈现利弊对比、或模式分析时，多用 1:1 或 1:1:1 的 `display: grid` 布局结构。列内部直接填充 `--bg-surface-light` 背景，构建直角软分区。
* 功能列在需要强调其优于其他选项时，可通过在顶部或左侧追加彩色的局部刻线（如 `border-left: 4px solid var(--accent)`）实现引导，而无需封装整个区块边框。

### 5.3 数据明细表格与矩阵排版 (Data Table & Grid Matrix)

商务风格的表格与矩阵要求极度透气、高度强调横向视觉流：
* **纵线彻底消除**：严禁在表格中出现任何交叉的垂直划分线（即 `border-right` 或 `border-left`）。
* **【物理贯穿横线铁律】 (Continuous Row Border Rule)**：这是大模型在处理多列复合表格时的排版红线。仅允许存在跨栏的横向底框线（如 `1px solid var(--grid-line)`）。当使用 CSS Grid 及 `gap` 实现列宽对齐时，**绝对禁止**将 `border-bottom` 挂载在独立的单元格子元素（Cell）上，因为 Grid `gap` 会无可避免地将其切断成廉价的破裂虚线。**必须**在每行外围强制包裹一层完整的行级 DOM 容器（例如 `<div class="row-container">` 或 `xxx-row`），并将边框或者背景色直接施加在整个行外围容器上，从而实现完美连贯无裂缝的贯穿分割线。
* **表头基座切断**：表头整体需覆以浅色底板（如 `var(--table-header)`），且其底端下边线必须采用重度粗实线（`2px solid var(--primary)`）以执行视觉压舱与内容截断。
* **首列锚定加粗**：表格或矩阵最左侧的第一维度列文本应设为纯黑（或 `--primary`）粗体（700），以充当使用者视线的横向检索固定锚。

### 5.4 概要金句框 (Summary & Focus Block)

* **强调框**：通常在页面主体下方呈现战略推演或金句。使用带有极弱透明彩度的底色（如 `rgba(0, 94, 184, 0.04)` 或 `.05`），并在 **左一侧** 添加粗实线（`4px solid var(--accent)`），彻底去除外围描边与圆角，呈现最清爽有力的导读重点。

### 5.5 封面页风格 (Cover Page)

封面是全套演示的视觉锚点，必须传达庄重、沉浸的品牌感。

* **底色**：整版铺满 `var(--primary)`，营造深沉沉浸感。可通过 `::before` 伪元素叠加一层极弱的斜角渐变（`--accent-light` 低透明度）增加空间层次。
* **标题区**：白色大号主标题 + 半透白副标题，垂直居中偏上。
* **元信息区**：标题下方承载报告类型与日期，白色系。
* **脚注区**：锁定底部，弱透白色，上方带极弱分割线。
* **禁止事项**：❌ 装饰性图片 ❌ 浅色卡片元素 ❌ 多段式彩色渐变。

## 6. 图标使用规范 (Icon Usage Guidelines)

商务风格中，图标的作用是**辅助结构化认知**而非装饰。必须保持极度克制、语义明确。

### 6.1 图标来源：Lucide Icons

统一使用 [Lucide Icons](https://lucide.dev) 作为图标来源。该库提供 1,500+ 精心设计的线条图标，风格简洁一致，与商务风格高度契合。

**引入方式**：通过 unpkg CDN 引入，在 HTML 中使用 `data-lucide` 属性声明图标：

```html
<!-- 在 </body> 前引入 -->
<script src="https://unpkg.com/lucide@latest"></script>
<script>lucide.createIcons();</script>

<!-- 在 HTML 中使用 -->
<i data-lucide="trending-up"></i>
<i data-lucide="check-circle"></i>
```

* **颜色**：通过 CSS `color` 属性控制，设置为 `var(--primary)` 或 `var(--accent)`。同一页面内同级图标统一颜色。
* **尺寸**：默认 24px，行内图标可缩至 `20px`（通过 CSS `width`/`height`），独立图标容器可放大至 `32px`。
* **适用场景**：列表项标识、区块标题辅助、流程步骤图标、状态标记等需要语义化图形的场景。

### 6.2 辅助图标类型

* **编号徽章**：深色直角小方块内嵌数字（`01`, `02`...）。用于步骤编号、维度序列。`--primary` 或 `--accent` 底色 + 白色文字，同一页面统一底色，与所属标题水平排列。
* **超大水印字符**：伪类 `::after` + 超大字号 + 极低透明度。用于背景层次增强（见 §4.2）。

### 6.3 通用约束

* 图标必须**语义匹配**内容，禁止随意装点。
* 图标必须伴随文字标签，不可无说明独立出现（水印伪类除外）。
* 同一页面内同级图标使用统一颜色（遵循 §4.3 色彩纪律）。
* ❌ 禁止引入 Font Awesome、Material Icons、Griddy Icons 等其他外部图标库。
